Step-by-Step Walkthrough

Learn how to use the Veritas Payload dataset effectively

01

Clone or Download

Start by cloning the repository or downloading the latest release from GitHub.

git clone https://github.com/HarlequinCS/Veritas_Payloads.git
02

Choose Your Format

The dataset is available as a single unified JSON file or organized into category folders.

Veritas_Payloads/ ├── owasp_top10_payloads.json # Unified (65,321 entries) ├── organize_payloads.py # Splitter tool └── payloads/ # Organized by category ├── injection_(xss)/query_parameter.json ├── injection_(sqli)/query_parameter.json └── injection_(lfi_rfi)/path_parameter.json
03

Understand the Schema

Each payload entry follows a consistent structure for easy integration.

{ "payload_id": "OWP-0001", "vulnerability_category": "Injection (SQLi)", "target_technology": "Generic", "injection_point": "query_parameter", "payload_string": "' OR '1'='1", "validation": { "expected_status_code": 200, "success_regex": "error|sql" } }
04

Load in Your Scanner

Use the organized files to feed your DAST or fuzzing tool. Each file targets a specific category and injection point.

import json with open("payloads/injection_(sqli)/query_parameter.json") as f: payloads = json.load(f) for p in payloads: print(f"{p['payload_id']}: {p['payload_string']}")
05

Re-organize Anytime

Run the included Python script to re-generate the organized folders after modifications.

python organize_payloads.py
06

Interpret Results

Each payload includes validation hints — expected status codes and success regex patterns — to help verify if a vulnerability was triggered.

SQL Injection Watch for: syntax error, SQL, unexpected 200 responses
XSS Watch for: <script>alert in response, DOM modifications
LFI/RFI Watch for: root:.*:0:0, boot.ini, file contents in response
Open Redirect Watch for: Location: header with 302 status
